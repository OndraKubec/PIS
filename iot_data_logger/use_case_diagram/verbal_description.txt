The IoT Data Logger system interacts with three primary actors: Admin, User, and Device. Each actor represents an external entity that communicates with the system in order to perform specific tasks related to data collection, management, and analysis.
The Admin actor represents a privileged system user responsible for the administration and configuration of the system. The administrator manages user accounts and connected IoT devices to ensure the system operates correctly. Through the User Management use case, the Admin can create, modify, or delete user accounts and manage their permissions. Through the Device Management use case, the Admin can register new IoT devices, configure their parameters, monitor their status, and remove devices that are no longer needed. These administrative functions allow the system infrastructure and access control to be maintained properly.
The User actor represents a standard user of the system who primarily interacts with collected data. The user accesses the Visualize Data use case, which allows them to view and analyze measurement data that has been collected by IoT devices. Within this functionality, the user can select a device, specify a time range, and display the data in graphical or tabular form. Before the system allows the user to access visualization features, the Authentication Check use case is performed. This ensures that the user is properly authenticated and authorized to access the requested data.
The Device actor represents an external IoT sensor or measurement unit connected to the system. Devices interact with the system through the Data Logging use case. In this process, devices send measured data to the system, which receives the data, validates it, and stores it in a database for later use. This stored data becomes available for visualization and further analysis by system users.
In addition to viewing data, users may also export the collected measurements using the Export Data use case. This functionality allows users to download selected datasets in formats such as CSV or JSON for further external processing or archiving. The export functionality is modeled as an extension of the Visualize Data use case, meaning that exporting data is an optional action that can be performed after or during the data visualization process.
Together, these actors and use cases define the core functionality of the IoT Data Logger system, which includes user administration, device configuration, automated data collection from IoT devices, secure access to stored data, data visualization, and optional export of measurement results.
